    FOR LOOPS

    • If we learn about the fucntion before so we can write a function that compute all numbers from 1 to n as below.
    
> compute_s_n <- function(n){
+     x <- 1:n
+     sum(x)
+ }
> compute_s_n(3)
[1] 6
> compute_s_n(100)
[1] 5050
> compute_s_n(2017)
[1] 2035153
    • Now, let’s say we want to compute this sum for various values of n. Say we want to compute it for 1, for 2 up to 25. So now we’re computing 25 sums. Do we write 25 lines of code, one from each n? No. That’s what loops are for.
    • Note that we are performing exactly the same task over and over again, except we’re changing n. For-loops let us define the range that our variable takes.
    • We can write a simple for loops as below.
    
> for(i in 1:5){
+     print(i)
+ }
[1] 1
[1] 2
[1] 3
[1] 4
[1] 5
    • We want to compute the sum for the values 1, 2, 3 up to 25. We can do as the code below
    
> m <- 25
> s_n <- vector(length = m)
> for(n in 1:m){
+     s_n[n] <- compute_s_n(n)
+ }
> s_n
 [1]   1   3   6  10  15  21  28  36  45  55  66  78  91 105 120 136 153 171 190 210 231 253 276 300 325
    • So the first thing I’m going to do is I’m going to create a vector, an empty vector where I’m going to store the results as I compute them. This is s_n.
    • Now we’re going to write a for loop. The index is n, and we will go from 1 through m, which is 25. So inside the loop, We are calling the function compute_s_n() with a value n. n is the value that’s changing from 1 through 25. And as we evaluate that, we assign it to the vector in the nth entry So we have defined the sum for values ranging from 1 through 25.
    • You can check the results with using formula as in the table below
    n s_n n(n+1)/2
    1 1 1
    2 3 3
    3 6 6
    4 10 10
    5 15 15
    6 21 21
    7 28 28
    8 36 36
    9 45 45
    10 55 55
    • you can use the plot and lines to see the results
    
> n <- 1:m
> plot(n, s_n)
> lines(n, n*(n+1)/2)

    SUMMARY

    • For-loops perform the same task over and over while changing the variable. They let us define the range that our variable takes, and then changes the value with each loop and evaluates the expression every time inside the loop.
    • The general form of a for-loop is: “For i in [some range], do operations”. This i changes across the range of values and the operations assume i is a value you’re interested in computing on.
    • At the end of the loop, the value of i is the last value of the range.
    
# creating a function that computes the sum of integers 1 through n
compute_s_n <- function(n){
  x <- 1:n
  sum(x)
}

# a very simple for-loop
for(i in 1:5){
  print(i)
}

# a for-loop for our summation
m <- 25
s_n <- vector(length = m) # create an empty vector
for(n in 1:m){
  s_n[n] <- compute_s_n(n)
}

# creating a plot for our summation function
n <- 1:m
plot(n, s_n)

# a table of values comparing our function to the summation formula
head(data.frame(s_n = s_n, formula = n*(n+1)/2))

# overlaying our function with the summation formula
plot(n, s_n)
lines(n, n*(n+1)/2)

    OTHER FUCNTIONS

    • Actually, we rarely use For Loops function in R. Functions that are typically used instead of for loops in R are apply, sapply, tapply, and mapply.These are part of what we call the apply family.
    • Other functions that are widely used are split, cut, quantile, reduce, identical, unique, and many others. There are many ways you can learn about this. And you should learn to use the web to find out about what they do and when to use them.
