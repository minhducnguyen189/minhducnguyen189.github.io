    CLASS

    • The function “class” helps us determine the type of an object. For example, if I define a as 2 and I look at its class, We will see it’s a numeric.
    
a <- 5
class(a)
"numberic"

class(ls)
"function"

    DATA FRAMES

    • The most common way of storing data sets in R is with data frames.

    • Conceptually, we can think of data frames as tables. Rows represent observations, and different variables are represented by different columns.

    • you can access example data by loading dslabs library. For exanoke you can load the murders data set by command data("murders").
    
library(dslabs)
data("murders")

    OBJECTS

    • So we know our data is stored in the object murders. But what is this object? How can we find out more about this object? The function str is useful for this. str stands for structure, and it shows us the structure of an object.
    
> str(murders)
'data.frame':	51 obs. of  5 variables:
 $ state     : chr  "Alabama" "Alaska" "Arizona" "Arkansas" ...
 $ abb       : chr  "AL" "AK" "AZ" "AR" ...
 $ region    : Factor w/ 4 levels "Northeast","South",..: 2 4 4 2 4 4 1 2 2 2 ...
 $ population: num  4779736 710231 6392017 2915918 37253956 ...
 $ total     : num  135 19 232 93 1257 ...
    • If you want to show some data you can use function head(<objectName>). So you will see 6 lines of data in your console.
    
head(murders)
      state abb region population total
1    Alabama  AL  South    4779736   135
2     Alaska  AK   West     710231    19
3    Arizona  AZ   West    6392017   232
4   Arkansas  AR  South    2915918    93

6   Colorado  CO   West    5029196    65

    ACCESS DATA IN OBJECT

    • For our analysis, we will need to access the different variables represented by columns. To access these variables, we use the dollar sign symbol $ . It’s called the accessor. For exmaple, we want to access the population in object murders we can use the command murders$population. The population is the name of column in object murders.

    VECTORS

    • When you get all data of a column by name so we received a list of data in the column, we call them are vectors. A single data is technically a vector. The function lenghth will help you count how many vectors you are.
    
> pop <- murders$population
> length(pop)
[1] 51

    CHARACTER VECTORS

    • We can store characters in R as well. Because variables also use character strings, we’re going to use quotes ("") to distinguish between variable names and character strings.

    LOGICAL VECTORS

    • Logical vectors are another type of vector. We don’t have those in this particular data frame, but we need to mention them. These must be either true or false. So we can create a logical vector by, for example, assigning 3 == 2
    
z <- 3 == 2
> z
[1] FALSE

    FACTOR

    • Factors are useful for storing what is called categorical data. The regions are categorical. There is only four categories. Each state is in one of these four. We can see the four levels in this particular factor using the function levels.
    
> levels(murders$region)
[1] "Northeast"     "South"         "North Central"
[4] "West"
    • Turns out that saving categorical data this way is more memory efficient. So in R, in the background, we store integers. We store the levels as integers. Technically, integers are smaller memory-wise than characters. So this is what makes it more efficient.
